# 画图题 SVG/PNG 扩展

只在同时满足以下条件时读取本文件：

- 题目要求画图、补图、作坐标图、几何图、流程图、示意图或函数图。
- 用户明确希望生成图片，或教程需要把图放入对应讲解文件。

如果用户没有要求生成图片，主教程只写文字解法，不加载本扩展。

## 目标

模仿例题的解法，生成可回查、可修改的 SVG，并渲染成 PNG 放入对应讲解文件。

默认产物：

```text
assets/drawings/Qxxx.svg
assets/drawings/Qxxx.png
```

教程中优先插入 PNG：

```markdown
![Qxxx 图解](assets/drawings/Qxxx.png)
```

必要时在图下补一句：

```markdown
SVG 源文件：`assets/drawings/Qxxx.svg`
```

## 工作流程

1. 先读懂例题图怎么画。
   - 识别坐标系、关键点、线段、箭头、标注、比例关系和解题步骤。
   - 总结例题的“作图套路”，不要只照抄图形外观。

2. 再读目标题。
   - 提取题目给定数据。
   - 明确需要画什么。
   - 如果缺少关键数据，标注“题目未给出，无法精确作图”，不要猜。

3. 生成 SVG。
   - SVG 是源文件，必须结构清楚、便于修改。
   - 优先使用 `<line>`、`<polyline>`、`<path>`、`<circle>`、`<text>`、`<marker>`。
   - 坐标、标签、箭头、辅助线都要和解题步骤一致。
   - 不做花哨装饰；图要服务于考试理解。

4. 渲染 PNG。
   - 优先使用本机可用工具把 SVG 转成 PNG。
   - 如果没有可用渲染器，保留 SVG，并在教程里说明 PNG 未生成原因。

5. 写入教程。
   - 图片放在对应题目的解析附近，不要集中堆到最后。
   - 图片前后要解释：图里每个关键元素对应题目哪一步。

## SVG 规范

- 使用白底或透明底，线条清晰。
- 图中文字尽量少，只放必要标签。
- 坐标图必须标出坐标轴和关键点。
- 几何图必须标出已知量、待求量和辅助线。
- 流程图必须用箭头表达先后顺序。
- 函数图如果不是精确绘制，必须标注“示意图”。

## PNG 渲染建议

可用哪个工具就用哪个：

```bash
rsvg-convert -o assets/drawings/Qxxx.png assets/drawings/Qxxx.svg
magick assets/drawings/Qxxx.svg assets/drawings/Qxxx.png
qlmanage -t -s 1200 -o assets/drawings assets/drawings/Qxxx.svg
```

如果渲染命令失败：

- 不要伪造 PNG。
- 保留 SVG。
- 在教程中写明“PNG 渲染失败，仅提供 SVG 源图”。

## 质量检查

每张图交付前检查：

- SVG 文件存在。
- PNG 文件存在，或明确说明未生成原因。
- 教程正文中已插入图片链接。
- 图中关键点、线段、箭头、标签和题目数据一致。
- 图的作法能对应例题套路。
- 图不是纯装饰，能帮助学生做题。

## 禁止

- 不要根据缺失数据猜图。
- 不要只生成“差不多”的漂亮图。
- 不要把图集中放在教程末尾。
- 不要用无法编辑的截图替代 SVG 源文件。
- 不要让图里的标签和正文变量名不一致。
