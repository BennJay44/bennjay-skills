# vibe-retrospective-tutor

总是 vibecoding 后感觉自己实际上没有什么收获？

需求做完了，项目也跑通了，提交记录看起来很漂亮。但关掉窗口之后，你可能还是说不清：这段代码为什么这样写？中间有哪些选择？如果下次没人带着做，自己能不能判断方向？

`vibe-retrospective-tutor` 就是为这个尴尬瞬间准备的中文复盘技能。它不会把一次编程任务简单压成“改了哪些文件”，而是整理成真正适合学习的复盘教程：改了什么，为什么这样做，有哪些取舍，中间踩了什么坑，怎么验证，以及下次遇到类似需求该怎么判断。

它不会在每次任务结束后自动打扰你。只有你明确说“复盘你是如何完成的”“给我写复盘教程”“讲讲这次怎么做的”这类话时才触发。

## 适合谁

- 正在用 Codex、Claude Code、Cursor 等智能体写代码的人
- 想在快速做需求的同时保留学习收益的人
- 需要把一次实现过程讲给自己、同学或团队听的人
- 不想只拿到“改了哪些文件”，而是想知道为什么这样改的人

## 它会输出什么

完整复盘通常包含：

1. 这次需求到底是什么
2. 改了哪些地方
3. 整体实现路线
4. 为什么选择这个技术或架构
5. 中间遇到的坑
6. 关键概念补课
7. 如何验证它真的工作
8. 可以继续练的小实验
9. 下次遇到类似需求怎么判断

小改动会走精简版，避免一件很小的事被写成一篇长文。

## 安装

### Codex

把本仓库放到项目级或用户级技能目录即可。

项目级安装：

下面的安装命令对应本仓库公开后的 `main` 分支。安装前请确认仓库已经公开可用。

```bash
mkdir -p .codex/skills
git clone https://github.com/BennJay44/vibe-retrospective-tutor.git .codex/skills/vibe-retrospective-tutor
```

用户级安装：

```bash
mkdir -p ~/.codex/skills
git clone https://github.com/BennJay44/vibe-retrospective-tutor.git ~/.codex/skills/vibe-retrospective-tutor
```

安装后重启 Codex，让它重新加载技能元数据。

### 其他智能体

如果你的工具支持 `SKILL.md` 格式，把 `SKILL.md` 放到对应的技能目录即可。也可以把 `SKILL.md` 内容作为自定义指令使用。

## 用法

完成一次编程任务后，直接说：

```text
复盘你是如何完成的
```

或者：

```text
给我写复盘教程
```

如果你只是说“总结一下刚才的改动”，技能会先澄清：你要普通总结，还是深入复盘教程。

## 设计原则

- 复盘只在用户主动要求时发生。
- 讲“为什么”，不只列“改了什么”。
- 技术判断必须有来源；没有来源就标注为推测，或者不写。
- 没跑过的测试不能说通过。
- 优先讲能迁移到其他项目的概念。
- 练习要安全、可撤回，不鼓励危险操作。

## 验证清单

发布或修改技能后，建议用下面几条输入做快速检查：

- 输入“复盘你是如何完成的”：应该触发完整复盘教程。
- 输入“总结一下刚才的改动”：应该先澄清是普通总结还是深入复盘。
- 单文件小改动：应该使用精简版结构。
- 多文件跨模块改动：应该使用完整结构，并解释技术取舍。
- 没有版本差异、对话里也没有文件线索：应该要求用户提供文件路径或版本差异，而不是编造复盘。
